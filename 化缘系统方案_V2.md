## 一、系统概述

**化缘系统**是玩家通过点击八卦区域进行类似抽奖的核心玩法。玩家选择区域后，触发随机事件，根据概率获得不同结果，最终盈亏以区域门槛金额为基准进行结算。

## 六、核心规则

### 6.1 准入规则
- **门槛验资**：玩家点击区域时，检查钱包余额是否≥门槛金额
- **余额不足**：区域按钮显示为灰色，无法点击，提示"香火钱不足"
- **不扣钱**：门槛仅作准入验证，点击时不扣钱

### 6.2 次数限制
- **全区累计**：每日20次，8个区域共用次数池
- **重置时间**：每日0点重置
- **无付费加次数**：人人平等，无VIP特权，无付费购买
- **无冷却时间**：可连续点击
- **后续调整**：跑数据后再决定是否调整次数

### 6.3 化缘流程
```
1. 玩家点击区域图标
2. 系统验资（余额≥门槛？）
3. 触发事件，弹出选择窗口
   - 前6区："虔诚礼拜，稳中求福" / "孤注一掷，逆天改命"
   - 后2区：同样显示选择（但不影响结果，隐藏规则）
4. 玩家选择
5. 系统根据选择调整概率/倍率（前6区）
6. 随机抽奖，确定结果
7. 前端展示动画（吉/凶）
8. 结算盈亏，更新钱包余额
9. 次数-1
```

### 6.4 结算规则
- **结算时机**：结果确定后才扣/加钱
- **计算公式**：
  ```
  最终余额 = 当前余额 + (门槛金额 × 收益倍数)
  ```
- **示例**：
  - 门槛100，结果JACKPOT：余额 +900
  - 门槛100，结果MISS：余额 -100

### 6.5 概率公示
- **前端不展示**：玩家看不到具体概率数字
- **仅展示吉/凶**：结果出来后显示"吉"或"凶"
- **后台可配置**：运营可根据数据调整概率表

## 七、连亏保护机制 ✨新增

### 7.1 规则
- 连续MISS（血本无归）**3次**后，下一次MISS概率**降低50%**
- 连续MISS **5次**后，下一次**保底NORMAL**（不亏不赚）
- 保护触发后重置计数

### 7.2 目的
- 防止新手连亏退游
- 不影响长期期望收益
- 给玩家"运气会回来"的感觉

### 配置表结构
```json
{
  "area_id": "tianlu",
  "name": "天禄",
  "trigram": "乾",
  "threshold": 100,
  "results": {
    "JACKPOT": {"prob": 0.02, "multiplier": 9},
    "BIG_WIN": {"prob": 0.10, "multiplier": 2},
    "NORMAL": {"prob": 0.50, "multiplier": 0},
    "SMALL_WIN": {"prob": 0.30, "multiplier": -0.7},
    "MISS": {"prob": 0.08, "multiplier": -1}
  }
}
```

### 安全注意
- **随机数**：使用加密安全的随机数生成器
- **防作弊**：后端计算结果，前端只展示动画
- **日志**：每次化缘记录日志（玩家ID、区域、选择、结果、盈亏）

## 十五、注意事项

1. **后2区（甘泽、流泉）不受选择影响**（隐藏规则，玩家不知道）
